Estructura XSLT

Una hoja de estilo XSL es un documento XML que define cómo transformar un archivo XML de entrada. No se escribe como un programa con instrucciones, sino como un conjunto de plantillas que indican qué hacer con cada parte del documento de entrada.

1. Estructura mínima​

Una hoja de estilo XSL contiene, al menos, los siguientes elementos:

  • La declaración XML

    <?xml version="1.0" encoding="UTF-8"?>

  • La raíz <xsl:stylesheet>
    Define el espacio de nombres y la versión de XSLT.

    <?xml version="1.0" encoding="UTF-8"?>
    <xsl:stylesheet xmlns:xsl="http://www.w3.org/1999/XSL/Transform" version="1.0">
      <!-- pantillas -->
    </xsl:stylesheet>

  • La salida deseada
    Se indica con <xsl:output>.

    <?xml version="1.0" encoding="UTF-8"?>
    <xsl:stylesheet xmlns:xsl="http://www.w3.org/1999/XSL/Transform" version="1.0">
      <xsl:output method="html" indent="yes"/>
      <!-- pantillas -->
    </xsl:stylesheet>

  • Una o varias plantillas
    Cada una transforma una parte del XML.

    <?xml version="1.0" encoding="UTF-8"?>
      <xsl:stylesheet xmlns:xsl="http://www.w3.org/1999/XSL/Transform" version="1.0">
      <xsl:output method="html" indent="yes"/>
      <xsl:template match="/">
        <!-- contenido generado -->
      </xsl:template>
      </xsl:stylesheet>

    El atributo match indica los elementos afectados por la plantilla y contiene una expresión XPath.

2. ¿Qué son las plantillas?​

Las plantillas (<xsl:template>) son bloques de código que definen cómo transformar los nodos del documento XML que cumplen una determinada condición. Esa condición se especifica con el atributo match, utilizando expresiones XPath. Las plantillas son el componente principal de XSLT: definen qué se genera y cuándo se genera durante la transformación.

¿Qué es una plantilla vacía?

Una plantilla vacía es una plantilla que coincide con ciertos nodos pero no genera ninguna salida. Sirve para ocultar nodos o interrumpir el comportamiento por defecto del procesador.

<xsl:template match="comentario"/>

o

<xsl:template match="comentario">
  <!-- no hace nada -->
</xsl:template>

Las plantillas vacía nos permiten:

  1. Evitar que se muestre el texto de ciertos nodos, como notas, comentarios, etc.
  2. Bloquear el comportamiento por defecto del procesador (mostrar el texto del nodo si no hay plantilla).
  3. Ayudan a limpiar la salida si solo se quieren procesar ciertos elementos del XML.

3. Añadir HTML dentro de una plantilla​

Una hoja de estilo XSLT permite mezclar etiquetas HTML y contenido dinámico extraído de un archivo XML.
Esto se consigue escribiendo directamente código HTML dentro de una plantilla y utilizando instrucciones XSLT como <xsl:value-of> o <xsl:for-each> para insertar los datos.

¿Cómo funciona?

En lugar de generar HTML desde cero con código en cada nodo, se puede escribir la estructura HTML directamente en la plantilla principal.
El procesador simplemente insertará los datos XML en los lugares definidos por el programador.

warning

En este caso escribimos HTML directamente dentro de la plantilla porque solo vamos a trabajar con salida en formato HTML. Sin embargo, lo más recomendable sería separar el diseño de la lógica si quisiéramos reutilizar la hoja de estilo para otros formatos (como texto, XML o PDF).

  • XML entrada
  • XSLT
  • HTML salida
<tienda>
  <catalogo>
    <producto>
      <nombre>Elixir de invisibilidad</nombre>
      <precio>80</precio>
    </producto>
  </catalogo>
</tienda>
<xsl:template match="/">
  <html>
    <head>
      <title>Catálogo de productos</title>
      <link rel="stylesheet" href="estilo.css"/>
    </head>
    <body>
      <h1>⭐ Tienda mágica ⭐</h1>
      <xsl:for-each select="tienda/catalogo/producto">
        <div class="producto">
          <p><strong><xsl:value-of select="nombre"/></strong></p>
          <p>Precio: <xsl:value-of select="precio"/> 💰</p>
        </div>
      </xsl:for-each>
    </body>
  </html>
</xsl:template>
<html>
  <head>
    <title>Catálogo de productos</title>
    <link rel="stylesheet" href="estilo.css">
  </head>
  <body>
    <h1>⭐ Tienda mágica ⭐</h1>
    <div class="producto">
      <p><strong>Elixir de invisibilidad</strong></p>
      <p>Precio: 80 💰</p>
    </div>
  </body>
</html>

4. Funcionamiento básico de XSLT​

Cuando se aplica una hoja de estilo XSLT a un documento XML, el procesador realiza una serie de pasos para transformar los datos. A continuación se detalla cómo funciona este proceso:

Paso 1: Buscar la plantilla inicial

Cuando se inicia la transformación, el procesador busca una plantilla que coincida con el nodo raíz del XML (/). Este nodo suele contener todos los demás datos, y es el punto de entrada de la transformación.

<xsl:template match="/">
  • XML entrada
  • XSLT
  • HTML salida
  <tienda>
    <catalogo>
      <producto>
        <nombre>Polvos de cuerno de unicornio</nombre>
        <precio>25</precio>
      </producto>
    </catalogo>
  </tienda>
   <xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
   <xsl:output method="html"/>
     <xsl:template match="/">
       <html>
         <body>
           <h1>Bienvenido a la tienda</h1>
         </body>
       </html>
     </xsl:template>
   </xsl:stylesheet>
    <html>
      <body>
        <h1>Bienvenido a la tienda</h1>
      </body>
    </html>

En este primer paso no se muestran los productos ni los datos del catálogo, ya que no se han aplicado plantillas adicionales ni se ha recorrido el árbol XML.

Paso 2. Aplicar plantilla correspondiente al nodo

Cuando el procesador encuentra un nodo que coincide con una plantilla (<xsl:template match="..."), ejecuta el contenido de esa plantilla.
Pero atención: no recorre automáticamente los hijos de ese nodo, a menos que se indique lo contrario con <xsl:apply-templates/>.

  • XML entrada
  • XSLT
  • HTML salida
<tienda>
  <catalogo>
    <producto>
      <nombre>Polvos de cuerno de unicornio</nombre>
      <precio>25</precio>
    </producto>
  </catalogo>
</tienda>
<xsl:stylesheet version="1.0"  xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
 <xsl:output method="html"/>
 <!-- Plantilla raíz -->
 <xsl:template match="/">
   <html>
     <body>
       <xsl:apply-templates select="tienda/catalogo"/>
     </body>
   </html>
 </xsl:template>
 <!-- Plantilla para catalogo -->
 <xsl:template match="catalogo">
   <h2>Catálogo cargado</h2>
 </xsl:template>
</xsl:stylesheet>
<html>
  <body>
    <h2>Catálogo cargado</h2>
  </body>
</html>

✅ El nodo <catalogo> ha sido procesado correctamente.
❌ Pero no se ha mostrado ningún producto, porque no hemos recorrido sus hijos ni definido plantillas para ellos.

Paso 3. Recorrer nodos hijos con <xsl:apply-templates/>

Para que el procesador XSLT recorra los nodos hijos de un elemento (por ejemplo, los <producto> dentro de <catalogo>), es necesario utilizar:

<xsl:apply-templates/>

Este elemento indica al procesador que continúe bajando por el árbol XML y aplique las plantillas correspondientes a cada hijo.

  • XML entrada
  • XSLT
  • HTML salida
  • XSLT una plantilla
<tienda>
  <catalogo>
    <producto>
      <nombre>Polvos de cuerno de unicornio</nombre>
      <precio>25</precio>
    </producto>
  </catalogo>
</tienda>
   <xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
    <xsl:output method="html" indent="yes"/>
    <!-- Plantilla raíz -->
    <xsl:template match="/">
      <html>
        <body>
          <xsl:apply-templates select="tienda/catalogo"/>
        </body>
      </html>
    </xsl:template>
    <!-- Plantilla para catalogo -->
    <xsl:template match="catalogo">
      <h2>Catálogo</h2>
      <div class="productos">
        <xsl:apply-templates/>
      </div>
    </xsl:template>
    <!-- Plantilla para producto -->
    <xsl:template match="producto">
      <div class="producto">
        <xsl:apply-templates/>
      </div>
    </xsl:template>
    <!-- Plantilla para nombre -->
    <xsl:template match="nombre">
      <p><strong>Nombre:</strong> <xsl:value-of select="."/></p>
    </xsl:template>
    <!-- Plantilla para precio -->
    <xsl:template match="precio">
      <p><strong>Precio:</strong> <xsl:value-of select="."/></p>
    </xsl:template>
  </xsl:stylesheet>
<html>
  <body>
    <h2>Catálogo</h2>
    <div class="productos">
      <div class="producto">
        <p><strong>Nombre:</strong> Polvos de cuerno de unicornio</p>
        <p><strong>Precio:</strong> 25</p>
      </div>
    </div>
  </body>
</html>
<xsl:stylesheet version="1.0"  xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
  <xsl:output method="html" indent="yes"/>
  <xsl:template match="/">
    <html>
      <body>
        <h2>Catálogo</h2>
        <div class="productos">
          <!-- Recorremos todos los productos desde la raíz -->
          <xsl:for-each select="tienda/catalogo/producto">
            <div class="producto">
              <p><strong>Nombre:</strong> <xsl:value-of select="nombre"/></p>
              <p><strong>Precio:</strong> <xsl:value-of select="precio"/></p>
            </div>
          </xsl:for-each>
        </div>
      </body>
    </html>
  </xsl:template>
</xsl:stylesheet>

✅ Gracias a <xsl:apply-templates/>, el procesador ha podido aplicar las plantillas a producto, nombre y precio, recorriendo el árbol XML completamente.

❌Si al recorrer los nodos con <xsl:apply-templates/> no existe una plantilla específica para alguno de ellos, el procesador incluye únicamente su contenido textual, sin etiquetas ni formato, en el doccumento de salida.

Paso 4. Finaliza la transformación y se genera el documento

Una vez que el procesador ha aplicado todas las plantillas necesarias y recorrido el árbol XML, se genera el documento de salida. Este documento puede ser de tipo html, xml o text, según lo que se indique en:

<xsl:output method="..."/>

Si no se indica nada, el valor por defecto es xml.

  • XML entrada
  • XSLT
  • HTML salida
<tienda>
  <catalogo>
    <producto>
      <nombre>Polvos de cuerno de unicornio</nombre>
      <precio>25</precio>
    </producto>
    <producto>
      <nombre>Colmillos de basilisco</nombre>
      <precio>60</precio>
    </producto>
  </catalogo>
</tienda>
  <xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
  <xsl:output method="html" indent="yes"/>
  <xsl:template match="/">
    <html>
      <head>
        <meta charset="UTF-8"/>
        <title>Catálogo de productos</title>
      </head>
      <body>
        <h1>Catálogo</h1>
        <xsl:for-each select="tienda/catalogo/producto">
          <div class="producto">
            <p><strong><xsl:value-of select="nombre"/></strong></p>
            <p>Precio: <xsl:value-of select="precio"/> 💰</p>
          </div>
        </xsl:for-each>
      </body>
    </html>
  </xsl:template>
  </xsl:stylesheet>
<html>
  <head>
    <meta charset="UTF-8">
    <title>Catálogo de productos</title>
  </head>
  <body>
    <h1>Catálogo</h1>
    <div class="producto">
      <p><strong>Polvos de cuerno de unicornio</strong></p>
      <p>Precio: 25 💰</p>
    </div>
    <div class="producto">
      <p><strong>Colmillos de basilisco</strong></p>
      <p>Precio: 60 💰</p>
    </div>
  </body>
</html>
  • El procesador ha recorrido todo el árbol XML desde la plantilla raíz.
  • Se ha generado un documento HTML bien formado.
  • Gracias a <xsl:output method="html"/>, el navegador puede interpretar correctamente el resultado.
