6.2 XPath

1. ¿Qué es XPath?​

XPath (XML Path Language) es un lenguaje que se utiliza para navegar por la estructura de un documento XML y localizar información dentro de él.

Permite seleccionar nodos, atributos o valores de texto de un archivo XML, siguiendo rutas similares a las de un sistema de archivos.

XPath sirve para:

  • Buscar y seleccionar elementos o atributos específicos dentro de un XML.
  • Filtrar información en función de condiciones.
  • Recorrer estructuras jerárquicas de datos XML.
  • Se utiliza como base en otras tecnologías, como XSLT, XQuery o XML Schema.

2. Estructura de nodos en XPath​

XPath interpreta un documento XML como un árbol de nodos, donde cada nodo representa una parte del contenido: elementos, atributos, textos, etc. Estos nodos están organizados de forma jerárquica, como si fueran ramas de un árbol.

Esta estructura jerárquica permite recorrer el documento fácilmente utilizando rutas, similares a las de un sistema de archivos. A través de estas rutas, XPath puede localizar con precisión cualquier nodo del XML, sin importar en qué parte del documento se encuentre.

🔸 Tipos de nodos en XPath​

Antes de poder utilizar XPath para localizar información dentro de un documento XML, es importante conocer los tipos de nodos con los que trabaja. XPath no solo accede a etiquetas, sino que también distingue entre elementos, atributos, texto y relaciones entre nodos (como padres, hijos o hermanos).

La siguiente tabla recoge los principales tipos de nodos que reconoce XPath:

Tipo de nodoDescripción
Nodo raízEs el nodo principal del documento. Desde él parten todos los demás.
Nodos descendientesSon los hijos, nietos, bisnietos... de un nodo dado.
Nodos ascendientesSon los padres, abuelos, etc., de un nodo.
Nodos hermanosSon nodos que comparten el mismo nodo padre.
Nodo elementoRepresenta una etiqueta XML, como <producto> o <cliente>.
Nodo atributoRepresenta atributos como id="pr01" dentro de una etiqueta.
Nodo de textoContiene el texto entre las etiquetas, como el nombre de un producto.

XPath utiliza una sintaxis sencilla basada en rutas para localizar nodos dentro de un documento XML.
Estas rutas se parecen a las de un sistema de archivos, como si navegaras por carpetas.

🔸 Separador / (barra simple)​

Se utiliza para ir directamente de un nodo padre a uno hijo.

Se utiliza para seleccionar nodos desde cualquier lugar del documento, sin importar su posición jerárquica.

  • Resultado
  producto/nombre
OJO

Desde las pruebas online no se obtiene resultado con rutas como producto/nombre,
porque se parte automáticamente desde el nodo raíz (/) y no sabe dónde está ese producto.

En otras palabras, no hay contexto definido, así que la ruta relativa no funciona.

🔸 ¿Qué es el contexto en XPath?​

En XPath, el contexto es el punto de partida desde el cual se evalúa una expresión. Es fundamental cuando se utilizan rutas relativas, ya que estas no comienzan desde la raíz del documento, sino desde un nodo concreto.

Por ejemplo, dentro de instrucciones como xsl:for-each o xsl:template, el contexto se establece previamente, y XPath evalúa la ruta desde ese nodo en lugar de desde el inicio del documento.

🔹 Si se usa una ruta relativa sin definir un contexto adecuado, XPath no sabrá desde dónde empezar la búsqueda y puede no devolver ningún resultado.

🔸 En cambio, si no se indica un contexto (como suele ocurrir en herramientas online o en pruebas simples), XPath asume como contexto el nodo raíz del documento.

🔸 ¿Qué devuelve XPath?​

XPath puede devolver distintos tipos de resultados, dependiendo de la expresión utilizada:

  • Nodos: si la expresión selecciona elementos, atributos o cualquier tipo de nodo, XPath devuelve uno o varios nodos del documento.

    /tienda/catalogo/producto

    Devuelve todos los nodos <producto> en esa ruta.

  • Contenido textual: si la expresión termina en un nodo de texto o utiliza la función text(), se devuelve el contenido dentro del nodo.

    /tienda/catalogo/producto/nombre/text()

    Devuelve solo los nombres de los productos (sin etiquetas).

  • Atributos: si se usa @, devuelve el valor de un atributo.

    /tienda/catalogo/producto/@id

    Devuelve los valores del atributo id de cada producto.

  • Números, booleanos o cadenas: si se utilizan funciones (como count(), contains() o string()), XPath puede devolver datos directamente, no nodos.

🔸 Selección de nodos por nombre​

Para seleccionar nodos específicos simplemente se indica su nombre.

        //producto

Devuelve todos los nodos producto del documento.

        //nombre

Devuelve todos los nodos nombre, estén donde estén (productos, clientes...).

        //clientes

Devuelve el nodo clientes, que contiene a todos los cliente.

🔸 Selección de atributos​

Para acceder a los atributos de un nodo se utiliza el símbolo @.

  • Consulta XPath
  • Resultado Text
  • Resultado Node
//producto[@id]

Devuelve todos los nodos producto que tienen un atributo @id. Si queremos el producto con un @id determinado:

//producto[@id="pr01"]
  1. Polvos de cuerno de unicornio Ingredientes raros 120.00
   <producto id="pr01">
       <nombre>Polvos de cuerno de unicornio </nombre>
       <categoria>Ingredientes raros </categoria>
       <precio>120.00 </precio>
   </producto>

🔸 Índices (posición)​

XPath permite acceder a nodos por su posición. El primer nodo es el número 1.

        //producto[3]

Devuelve el tercer nodo producto encontrado en el documento.

4. Condiciones en XPath​

XPath permite aplicar condiciones para filtrar nodos y obtener solo aquellos que cumplan ciertos criterios.
Estas condiciones se colocan entre corchetes [ ] y se pueden aplicar sobre:

  • Texto de nodos
  • Atributos
  • Posición dentro del conjunto
  • Funciones de texto o posición
¿Qué significa . en XPath?

En XPath, el punto . se refiere al nodo actual.
Cuando se usa dentro de una condición como [. != 'valor'], indica que estamos evaluando el contenido del nodo actual (normalmente, su texto).

categoria[. != 'Plantas mágicas']

Selecciona todos los elementos categoria cuyo contenido textual no sea "Plantas mágicas".

El . hace referencia al nodo actual categoria y su contenido.

A continuación se explican los distintos tipos de condiciones, de menor a mayor complejidad.

🔸 Comparaciones básicas​

Permiten seleccionar nodos cuyo contenido coincida exactamente con un valor.

  • Por contenido textual:

    //producto[nombre = 'Raíz de mandrágora']

    ✔️ Devuelve el producto cuyo nombre coincide exactamente.

  • Por atributo:

    //cliente[@id = 'c002']

    ✔️ Devuelve el cliente con id igual a "c002".

  • Por número:

    //producto[precio > 100]

    ✔️ Devuelve productos con precio mayor que 100.

🔸 Condiciones con funciones de texto​

Permiten realizar comparaciones más flexibles sobre cadenas de texto, útiles para búsquedas parciales o transformación de cadenas.

Función¿Qué hace?EjemploResultado esperado
string()Convierte un nodo en texto.//producto[string(nombre) = 'Raíz de mandrágora']Producto con ese nombre exacto.
contains()Comprueba si una cadena contiene otra.//producto[contains(nombre, 'dragón')]Productos cuyo nombre contiene “dragón”.
starts-with()Comprueba si comienza con cierto texto.//producto[starts-with(nombre, 'Polvos')]“Polvos de cuerno de unicornio”.
ends-with()*Comprueba si termina en cierto texto (XPath 2.0+).//producto[ends-with(nombre, 'Oscuridad')]Productos cuyo nombre termina en “Oscuridad”.
substring()Extrae parte del texto.//producto[substring(nombre, 1, 6) = 'Poción']Nombres que empiezan por “Poción”.
substring-before()Extrae texto antes de una subcadena.//producto[substring-before(nombre, ' de') = 'Raíz']“Raíz de mandrágora”.
substring-after()Extrae texto después de una subcadena.//producto[substring-after(nombre, ' de') = 'mandrágora']“Raíz de mandrágora”.
string-length()Longitud del texto.//producto[string-length(nombre) > 20]Nombres largos como “Agua bendita del lago helado”.
normalize-space()Elimina espacios duplicados y bordes.//producto[normalize-space(nombre) = 'Flor de fuego nocturno']Evita errores por espacios de más.
translate()Sustituye caracteres (útil para comparar sin tildes o mayúsculas).//producto[translate(nombre, 'aeiou', 'AEIOU') = 'POCION DE FUEGO']Compara texto transformado (sin distinción de mayúsculas).

Nota: algunas funciones como ends-with() requieren XPath 2.0.

🔸 Condiciones por posición​

Permiten filtrar nodos según su posición dentro del conjunto.

  • Primer nodo:

    //producto[position() = 1]

  • Segundo nodo:

    //producto[2]

  • Último nodo:

    //producto[last()]

  • Nodos antes de cierto punto:

    //producto[position() < 4]

  • Penúltimo nodo:

    //producto[last()-1]

🔸 Otras funciones útiles​

Además de las expresiones y funciones básicas explicadas en los apartados anteriores, XPath 1.0 ofrece otras funciones muy útiles para realizar cálculos, convertir datos, formatear resultados o trabajar con el contexto de evaluación. A continuación se recopilan las más comunes agrupadas por su finalidad:

CategoríaFunciónDescripciónEjemplo
Agregacióncount(nodoset)Devuelve cuántos nodos hay en un conjunto.count(//producto[@destacado='sí'])
Agregaciónsum(nodoset)Suma los valores numéricos de un conjunto de nodos.sum(//linea/@cantidad)
Formateoformat-number(num, patrón)Formatea un número según el patrón indicado. Es muy útil para mostrar precios, porcentajes o decimales con una cantidad fija de dígitos.format-number(3.1416, '#.00') → 3.14
Conversiónnumber(valor)Convierte un valor en número para poder operar con él.number(precio)
Conversiónstring(valor)Convierte cualquier valor en texto.string(123.45) → "123.45"
Conversiónboolean(valor)Convierte un valor en true o false según su contenido.boolean('') → false
Contextocurrent()Devuelve el nodo del contexto externo (por ejemplo, en bucles anidados).sum(//linea[@producto = current()/@id]/@cantidad)
current()

La función current() se utiliza en XPath 1.0 dentro de expresiones complejas para recuperar el nodo del contexto externo.
Es especialmente útil en bucles anidados (for-each) o filtros cuando se necesita seguir accediendo al nodo original del que se partió.

CONVERSIONES

XPath 1.0 realiza conversiones automáticas entre tipos cuando es posible.
Por ejemplo, en comparaciones y operaciones matemáticas, una cadena que contiene un número puede interpretarse como número sin usar number().
Aun así, usar number() explícitamente puede evitar errores en transformaciones más complejas.

warning

En XPath 1.0 no existen funciones como avg() o max().
Para calcular medias, mínimos o máximos es necesario combinar funciones como sum(), count() y position().
